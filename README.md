### `> whoami`

I'm Tejasa — a software engineer who builds across the full stack and actually cares about all of it.

Backend systems that don't fall over under load. Databases that stay sane at scale. Cloud infra that's boring in the best possible way. Frontends that feel like someone thought about the person using them.

That's the foundation — built across real fintech and SaaS products where the stakes are real and users notice when something breaks.

And lately? I've been extending that into AI engineering — not as a pivot, but as a natural next layer. Building reliable, useful AI features on top of solid backend foundations is where the most interesting problems live right now, and I want to be building them.

### things I've learned by breaking them first

> The best products aren't built by people who follow specs — they're built by people who can't stop thinking about the user on the other end.

I've shipped distributed Go services, Python pipelines, React frontends, AWS infrastructure, and everything in between. But what I actually enjoy is figuring out *why* something needs to exist before figuring out *how* to build it — and then making sure it holds up when real people actually use it.
